# Task 2 – Phase 1 Capabilities

## Included Capabilities

| Capability                               | Description                                                                                                                                                                                                                                                                   | Justification                                                                                                                                                                                                                                   |
| ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Identity Verification**                | Implement Multi-Factor Authentication (MFA) that matches customers to existing records. If MFA fails three times, the account should be temporarily locked, a contact number for an agent should be displayed, and the case should be flagged to Gareth's team for follow-up. | Mandatory under UK FCA security and data protection requirements. Robust identity verification is essential to protect customer data and maintain organisational credibility.                                                                   |
| **Account Summary Dashboard**            | Provide a clean and intuitive dashboard displaying key information such as current balance, outstanding debt, recent activity, and account status.                                                                                                                            | Enables customers to access information independently, reducing demand on agents and supporting **OPP-01 (Automate Status Checks)**.                                                                                                            |
| **Promise-to-Pay Setup**                 | Allow customers to create repayment plans through a self-service portal. Options include repayment by a specified date or through an instalment plan. Repayment amounts should be checked against monthly income, with unaffordable plans escalated to an agent.              | Reduces agent workload by enabling customers to manage straightforward repayment arrangements themselves, while ensuring affordability concerns are reviewed by staff. Supports Gareth's objectives.                                            |
| **Automated Workflow Alerts**            | Automatically detect when a payment has not been made by the agreed deadline and flag the case for agent follow-up.                                                                                                                                                           | Supports **OPP-05 (Automate Scheduling of Follow-Ups with Agents)** and helps reduce revenue loss from missed repayments.                                                                                                                       |
| **Data Capture, Auditing and Reporting** | Record customer actions through audit logs and feed data into an automated reporting dashboard.                                                                                                                                                                               | Essential for regulatory compliance, auditability, and risk management. Although **OPP-03 (Automate Analytics and End-of-Month Reporting)** may have a negative ROI, auditing remains a business necessity and aligns with Daniel's objectives. |

## Excluded Capabilities

| Capability                                                            | Description                                                                                                                                                                                 | Justification                                                                                                                                                                            |
| --------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Direct Database Integration via APIs**                              | The portal will not write directly to the legacy database. Instead, data will be collected in CSV files, transferred nightly via automated SFTP, and processed by separate update routines. | Direct integration with the 20-year-old legacy database presents significant technical and operational risk, including potential disruption to critical systems. Relates to **OPP-04**.  |
| **Automatic Entry of Delinquent Accounts into the Recovery Database** | Customers will not be automatically transferred into the recovery database immediately after becoming delinquent.                                                                           | This functionality depends on direct interaction with the legacy database and therefore carries the same risks identified in **OPP-04**. Deferred from Phase 1 and linked to **OPP-02**. |
| **Advanced Case Routing**                                             | Highly sensitive or vulnerable customer cases will be directed to human agents. The portal will identify such cases and provide customers with the appropriate contact information.         | Human judgement is required for complex and sensitive situations. Automated handling may fail to recognise vulnerabilities or appropriately address customer needs.                      |
